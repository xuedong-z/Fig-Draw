# Origin (OriginLab) SVG export — reverse-engineered rules

How SciCompose recognizes graphs exported as **SVG** from OriginLab. Reverse-engineered
from 15 real chart types generated headlessly via Origin's COM automation server
(Origin 2025). There is **no public XML schema** for the `olab` namespace, but the
`olab:scope` attribute is a serialization of Origin's documented graph object model,
which makes recognition rule-based and general across chart types.

Implemented in `frontend/lib/svg/parser.ts` (`isOriginSvg`, `normalizeOriginSvg`,
`originRoleFromHint`, `splitTiledOriginLayers`).

## Detection

A file is treated as an Origin export if it contains `originlab` (the `xmlns:olab`
URL) or `Generated by Origin`. Every drawable element then carries **both**:

- `class="Layer0 Layer Axis0 Axis SubAxis0 SubAxis Tick Major"`
- `olab:scope="Layer0.Axis0.SubAxis0.Tick-Major"`

The scope is a path through `Page → Layer{N} → {Axis{N}.SubAxis{N}.<part> | Plot{N} | (bare)}`.

## The complete scope vocabulary (verified, 7 patterns — nothing else exists)

| `olab:scope` pattern | Tag(s) | Fill / stroke | Role |
| --- | --- | --- | --- |
| `Layer{N}.Axis{N}.SubAxis{N}.Line` | `polyline` | black stroke, `fill:none` | **axis** (spine) |
| `…SubAxis{N}.Tick-Major` | `polyline` | black stroke | **tick** |
| `…SubAxis{N}.Tick-Minor` | `polyline` | black stroke | **tick** |
| `…SubAxis{N}.TickLabel-Major` | `text` | black fill | **text-tick** (numeric **and** category names) |
| `…SubAxis{N}.Title` | `text` | black fill; Y titles rotated via `matrix(...)` | **text-axis** |
| `Layer{N}.Plot{N}` | `polyline` / `polygon` / `rect` | colored | **data** / **scatter** (see below) |
| `Layer{N}` (bare) | `polyline` / `polygon` / `rect` / `text` | mixed | **legend** / **text-legend** |

Notes confirmed across all samples:

- **Chart type is NOT in the scope.** A line, scatter, bar/column, area — all data is
  just `Plot{N}`. Distinguish by geometry, not scope.
- `TickLabel-Minor`, `Grid`, `Break`, `Legend` tokens **never appear**.
- Category-axis labels (`"Alpha"`, `"Beta"`, …) use the **same** `TickLabel-Major`
  scope as numeric labels → always `text-tick`.

### Data sub-typing (`Plot{N}`), by tag + geometry
- `polyline` with `fill:none` → **data** (the connecting line; one multi-point line, or
  many 2-point segments).
- `polygon`/`rect`/`circle` with a colored `fill`:
  - **bar/column/area**: anchored to the axis baseline, value-dependent size → **data**.
  - **scatter symbol**: small fixed-size marker, free-floating → **scatter**.
  - The robust discriminator is "two vertices sit on the axis baseline" (bar) vs
    free-floating fixed size (marker). SciCompose approximates with a size test
    (small in *both* dimensions ⇒ marker), which works once geometry is measured 1:1.

### Legend (bare `Layer{N}`)
A legend is a cluster of bare-`Layer{N}` elements: a frame (4 `polyline` segments),
colored **swatches** (`polygon`/`rect`) whose fill/stroke are **byte-identical to the
`Plot{N}` series colors**, and entry `text`. SciCompose roles them **legend** /
**text-legend** so the swatches never inflate the data-series count, and
`pairLegends` recolors them with their series by color match.

## Coordinate normalization (`normalizeOriginSvg`)
- **Nested `<svg>`**: some exports wrap all geometry in an inner `<svg>` with a huge
  viewBox — flattened (adopt the inner viewBox, hoist children).
- **Width/height are in inches** (≈6× off the viewBox) — pinned to the viewBox so
  `getBBox·getCTM` measures 1:1 (true for nested **and** non-nested exports).
- **Plot-clip mask** stripped (an un-scaled mask would hide everything after the bake).
- **Rotated `matrix()` titles** → `x/y` + `rotate()`; subscript title fragments are
  merged into one `<text>` with `baseline-shift`.

## Multi-layer graphs (`splitTiledOriginLayers`)
Origin encodes layering purely in the scope prefix (`Layer{N}.…`), not via DOM
containers. Each layer is a **complete, independent plot region** (its own
`Axis0/Axis1/Plot0`).

- **Tiled / stacked panels** (layers occupy disjoint regions): split into **one panel
  per layer** — partition elements by their `Layer{N}` prefix, crop each panel's
  viewBox to that layer's bounding box.
- **Overlaid double-Y** (layers share one region, e.g. blue/gray dual axes): kept as a
  **single panel**. Detected by bounding-box overlap (>25% ⇒ overlaid).

## Known Origin export limitations (not recognizer bugs)
- **Grid lines are not exported** to SVG (no grid geometry, no `Grid` token) — must be
  synthesized from tick positions if needed.
- **Box plots, stacked columns, error-bar whiskers** export **without their data
  glyphs** — the SVG contains only axes/labels/legend. These types don't round-trip.
- **Log axes**: tick labels are decades (`1E-8 … 1000`) and the pixel position is
  linear in `log10(value)`, not in `value`. Pixel-based features (recolor, figsize,
  trim) still work; any value↔pixel calibration must fit in log space.
